# AI 에이전트 작업 로그

이 파일은 AI 에이전트가 수행한 개발 및 리팩토링 작업을 문서화하고, AI 에이전트의 행동 가이드라인을 정의합니다.

### 작업 관리

- 모든 작업은 시작 전에 `works/TODO.md` 파일에 기록됩니다.
- 작업이 완료되면 `works/TODO.md` 파일의 해당 항목을 완료(`[x]`)로 표시하고, `works/TODO.md`의 "완료된 일" 섹션에 `docs/DONE.md`의 Phase 구성을 참고하여 Phase별로 그룹화하여 기록합니다.
- **AI 에이전트는 `works/DONE.md` 파일을 직접 수정하지 않으며, 완료된 작업의 최종 이동 및 아카이빙은 사용자가 수행합니다.**
- **날짜 기록 주의**: 작업 로그에 날짜를 기록할 때는, **실제 작업이 수행된 날짜**를 `(YYYY-MM-DD)` 형식으로 기록합니다. 단, 과거 작업에 대한 기록을 추가할 경우, 프로젝트의 기존 작업 일지에 나타나는 시간적 배경을 확인하고 일관성을 유지할 수 있습니다.
- **테스트 시나리오 유지**: `works/issues` 디렉토리 내의 이슈 문서에 정의된 "테스트 시나리오"는 해당 기능의 완료 여부와 관계없이 유지되어야 합니다.

### 핵심 개발 원칙

- **코드, 테스트, 문서의 동기화**: 소스 코드를 수정할 때는 반드시 관련된 **테스트 코드**와 **문서(README, USAGE 등)**도 함께 업데이트해야 합니다. 모든 변경사항은 이 세 가지 요소에 일관성 있게 반영되어야 합니다.

### AI 에이전트 초기화 작업

AI 에이전트는 채팅 세션을 시작할 때 다음 파일들을 읽고 숙지해야 합니다:

- `README.md`
- `AGENTS.md`
- `docs` 디렉토리의 모든 마크다운 파일
- `works` 디렉토리의 모든 마크다운 파일
### 사용자 상호작용 가이드

- **언어**: 모든 응답은 반드시 한국어로 작성해야 합니다.
- **어조 및 스타일**: 동료 개발자와 페어 프로그래밍을 하는 것처럼, 전문적이면서도 친근하고 협력적인 어조를 사용해주세요.
